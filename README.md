## How I work

```text
Understand the problem
  -> define constraints and acceptance criteria
  -> design the data and interaction flow
  -> implement in reviewable increments
  -> test behavior and edge cases
  -> document the decision
```

- Product context before implementation.
- Clear boundaries between interface, service, and data concerns.
- AI assistance as an accelerator, never as the final authority.
- Maintainability, security, and user feedback treated as product features.
